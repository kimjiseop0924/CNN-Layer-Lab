테스트 샘플 수를 바꾸면 정확도가 얼마나 달라질까?

📍 핵심 관찰
테스트 10개로 평가하면 epoch 1~4 동안 점수가 계속 70% → 1개 단위로만 점수가 바뀌어서 그 사이 변화가 안 보임
마지막 epoch에서 90%로 점프한 것도 알고 보면 한 문제 더 맞춘 것뿐 → 모델이 갑자기 좋아진 게 아님
같은 모델 epoch 5인데 테스트 10개로 보면 90%, 1000개로 보면 82.8% → 측정 방법이 다르다는 이유만으로 7.2%p 차이 발생
테스트 1000개는 54 → 72 → 81 → 82.5 → 82.8로 부드럽게 올라감 → 진짜 학습 곡선이 보임
📌 결론
실험끼리 비교하려면 테스트는 최소 1,000개로 해야 한다. 그보다 적으면 점수가 7%p 넘게 흔들려서, 모델이 좋아진 건지 그냥 운이 좋았던 건지 알 수 없다.
EXPERIMENT 02

학습 데이터가 많아지면 정확도는 얼마나 오를까?

고정 테스트 1000, Conv 2  ·  변수 학습 128 / 512 / 2048
Epoch학습 128학습 512학습 2048
014.8%16.4%9.5%
115.3%25.5%50.4%
213.9%31.9%73.3%
318.6%40.2%81.1%
424.1%51.1%83.5%

📍 핵심 관찰
데이터를 4배 늘릴 때마다 점수가 거의 똑같이 +30%p씩 오름 → 두 배 늘렸다고 두 배 좋아지는 게 아니다
학습 128개는 끝까지 학습해도 25.7%에서 멈춤 → 데이터가 부족하면 오래 학습시켜도 소용없다
학습 2048개는 1 epoch에 벌써 50%, 5 epoch에 86% → 데이터가 많으면 시작부터 다르다
데이터를 4배 늘렸을 때 점수 변화 (Epoch 5 기준)
128 → 512 (×4)+29.7 %p
512 → 2048 (×4)+31.0 %p
📌 결론
데이터 4배 ≒ 점수 +30%p. 그런데 다음 30%p를 또 얻으려면 또 4배가 필요하다. 즉, 점수를 올리는 가장 확실한 방법은 데이터를 많이 모으는 것이고, 데이터가 적을 땐 오래 학습시켜도 한계가 있다.
